# Software Requirements Specification (SRS) - LexiLingo

## 1. Introduction
**Project Name:** LexiLingo  
**Purpose:** A mobile application for learning English, featuring AI-powered tutoring, course management, vocabulary building, and progress tracking.  
**Target Audience:** Individuals seeking to improve their English proficiency through a mobile-first experience.

## 2. Tech Stack
- **Framework:** Flutter (Dart)
- **Architecture:** MVVM or Clean Architecture (Provider/Riverpod/Bloc)
- **Database:** Local Storage (SQLite via `sqflite` or Hive) for offline capabilities.
- **Authentication:** Google Sign-In (`google_sign_in`, `firebase_auth` optional but recommended for ID consistency).
- **AI Integration:** Google Gemini API (or similar LLM API) for chat and grammar correction.
- **Notifications:** Local Notifications (`flutter_local_notifications`).

## 3. User Roles
- **Learner:** The primary user who logs in, takes courses, chats with AI, and tracks progress.

## 4. Functional Requirements

### 4.1 Authentication
- **FR-01:** User shall be able to log in using their Google account.
- **FR-02:** User session shall be persisted.
- **FR-03:** User shall be able to log out.

### 4.2 Home Dashboard
- **FR-04:** Application shall display a dashboard upon login.
- **FR-05:** Dashboard shall show current learning streak (days).
- **FR-06:** Dashboard shall show daily goals and progress.
- **FR-07:** Dashboard shall provide quick access to "Continue Learning" (last accessed course/lesson).

### 4.3 AI Chat Tutor
- **FR-08:** User shall be able to initiate a chat session with an AI Tutor.
- **FR-09:** AI shall capability to correct user's grammar in real-time or upon request.
- **FR-10:** AI shall engage in conversation (roleplay scenarios).
- **FR-11:** Chat history shall be saved locally.
- **FR-12:** AI should be integrated into other parts of the app (e.g., explaining vocabulary words).

### 4.4 Learning Courses
- **FR-13:** Application shall display a list of available courses/levels (e.g., Beginner, Intermediate).
- **FR-14:** User can select a course to view lessons.
- **FR-15:** System tracks completion status of lessons.

### 4.5 Vocabulary Library
- **FR-16:** User shall have a personal vocabulary list.
- **FR-17:** User can add new words manually or from lessons/chat.
- **FR-18:** User can view definitions (possibly generated by AI).

### 4.6 User Profile & Statistics
- **FR-19:** User can view their profile (Avatar, Name from Google).
- **FR-20:** User can view detailed statistics (Total words learned, hours spent, lessons completed).

### 4.7 Notifications
- **FR-21:** User can set a daily reminder time for studying.
- **FR-22:** System shall send a local push notification at the scheduled time.

## 5. Non-Functional Requirements
- **NFR-01:** **Performance:** The app should load screens within 2 seconds.
- **NFR-02:** **Offline Mode:** Core content (downloaded lessons, history) should be accessible without internet (AI features will require internet).
- **NFR-03:** **UI/UX:** The interface must follow the design provided in the HTML mockups (Clean, Light/Dark mode support).

## 6. Database Schema (Conceptual - SQLite)
- **Table `User`**: id, name, email, avatar_url, join_date.
- **Table `Courses`**: id, title, description, level, total_lessons.
- **Table `Lessons`**: id, course_id, title, status (locked/open/completed).
- **Table `Vocabulary`**: id, word, definition, learned_date.
- **Table `ChatMessages`**: id, role (user/ai), content, timestamp.
- **Table `Settings`**: notification_time, theme_preference.
